# Lanshare 高级感前端设计规范

本文档定义 Lanshare 课堂平台的前端设计语言，优先服务登录后首页、课堂工作台、教学管理和移动端重复使用场景。目标不是把界面做得更“炫”，而是在清晰、快速、可靠的基础上，让页面更有秩序感、呼吸感和被照顾的交互细节。

## 设计判断

- Lyssna: 设计师调研显示微交互、动画与 accessibility-first 设计持续提升影响力。https://www.lyssna.com/blog/ux-design-trends/

## 核心原则

1. 内容先行：每个首屏模块只回答一个问题，例如“我现在有哪些课堂”“哪些事项需要处理”“下一步去哪里”。
2. 留白有方向：留白用来分组和引导视线，不用来制造空旷。关键操作周围留出更大的触控与视觉缓冲。
3. 高级感来自克制：少用重渐变、大阴影和装饰图形。用高质量中性色、细边框、低透明层和准确的字体层级建立质感。
4. 动画服务理解：动效只表达进入、聚焦、选择、完成和状态变化。禁止滚动劫持和让用户等待装饰动画。
5. 移动端不是缩小版桌面：移动端优先呈现课堂列表、搜索筛选、待处理事项；辅助信息顺序后移，触控目标不小于 44px。
6. 功能不可被美学牺牲：已有入口、状态、空态、权限差异、搜索筛选、消息反馈、日历和安全信息必须保留可见或可达。

## 视觉系统

### 色彩

- 背景：以 warm gray / slate gray / soft mint 组成低饱和底色，避免单一蓝紫主导。
- 主色：学生端使用 indigo 作为行动色，教师端使用 emerald 作为行动色。
- 辅色：amber 用于提醒和时间感，cyan 用于轻量进度反馈，red 仅用于危险或未读风险。
- 文字：标题使用 `#0f172a`，正文使用 `#475569`，辅助信息使用 `#64748b` 或更浅灰阶。

### 层级

- 页面级容器不做厚重卡片，使用约束宽度、留白和轻微背景带区分。
- 重复项目才使用卡片，例如课堂卡、统计卡、快捷入口卡。
- 卡片圆角控制在 8px 左右，避免所有内容都变成圆润漂浮块。
- 阴影以“贴近纸面”为原则，默认轻阴影，悬停时只增加一点高度感。

### 字体

- 中文环境沿用现有系统字体栈。
- 首页 H1 桌面约 40px，移动端约 30px。不要使用 viewport 宽度直接缩放字体。
- 卡片标题 17px 到 20px，正文 14px 到 16px。
- `letter-spacing` 保持 0，避免中文界面出现不自然字距。

## 交互与动画

- 进入动画：列表项淡入并轻微上移，单项延迟不超过 80ms，整体不超过 480ms。
- 选择反馈：筛选项、课堂卡、快捷入口在 hover/focus 时使用 1px 位移、边框加深和轻阴影。
- 状态反馈：搜索结果数量、空态、未读提醒保持即时更新。
- 触控端：禁用依赖 hover 才能发现的信息；筛选条可横向滚动，卡片改为单列。
- 减少动态：尊重 `prefers-reduced-motion`，关闭入场位移和非必要过渡。

## 响应式规则

- `>= 1280px`: 首屏使用左右结构，课堂列表为主内容，右侧为快捷入口和动态。
- `960px - 1279px`: 首页主信息垂直重排，课堂列表优先，辅助面板后置。
- `640px - 959px`: 课堂卡单列，筛选和搜索上下排列，学期日历保持横向可滚动。
- `< 640px`: 去掉非必要导航文字，触控区域加大，卡片信息密度降低，避免按钮文字挤压。

## 首页改造核对清单

- 保留：课堂列表、进入课堂、筛选、搜索、清除筛选、空态、学期日历、快捷操作、关注事项、最近动态、登录安全、消息中心、博客、反馈、个人中心、登出、教师开设新课堂。
- 优化：首屏信息层级、移动端按钮换行、筛选横向滚动、卡片触控尺寸、动画节奏、色彩克制、无障碍焦点样式。
- 验证：Jinja 模板可解析，`dashboard.js` 语法通过，CSS 无明显语法错误，桌面和移动视口不出现横向溢出。

## 学期日历与待办

- 待办归属：首页“学期日历”承载跨课堂待办，课堂页不展示“教学日历与本周待办”面板。
- 周次对齐：待办以甘特条出现在日历底部的“第 x 周”列内，和每周日期网格保持同一横向滚动上下文。
- 角色差异：教师端显示课程安排与课堂任务截止；学生端显示课程安排、作业考试截止、个人试炼和自定义待办。
- 交互：点击甘特条展开详情，保留进入课堂或任务的跳转；学生可在学期日历内新增、完成、删除个人待办。
- 移动端：日历保持横向拖动浏览，详情区纵向堆叠，新增待办表单一列排列。

## 课堂页改造核对清单

- 保留：课程进度时间轴、课次材料入口、作业与考试入口、学习进度/教师学习概览、课程材料、课堂资源、文件详情、讨论区、图片/表情/引用、AI 助手、消息中心、反馈、个人中心、学生修改密码。
- 区分：教师端用 emerald 作为行动色，强调班级概览与管理入口；学生端用 indigo 作为行动色，强调学习进度、破境试炼和提交状态。
- 结构：课堂页以课程进度为首屏核心，其他模块保持原顺序和可达性；不把首页级的跨课堂待办塞进课堂工作区。
- 质感：使用轻边框、低透明白色表面和小幅 hover/focus 反馈；重复卡片圆角约 8px，避免装饰性大块漂浮。
- 移动端：导航和操作区允许换行，主内容单列，按钮宽度随容器收缩，悬浮快捷导航保持独立边距，不覆盖主要内容。
